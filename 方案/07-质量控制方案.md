# 7. 质量控制方案

## 7.1 遵循的标准

| 类别 | 标准 / 规范 |
|---|---|
| 主标准 | GJB 438C-2021《军用软件开发文档通用要求》 |
| 编码 | Google C++ Style + 内部补充规范 |
| 文档 | 甲方提供的模板（设计说明、用户手册） |
| 单元测试 | QtTest 框架；测试用例与被测代码一一对应 |

## 7.2 设计约束控制

| 项 | 控制方法 |
|---|---|
| 设计、投图、调试、生产 | 严格按技术协议书 / 技术要求执行；变更需经评审确认 |
| 技术状态 | 按 GJB 438C-2021 控制基线；每阶段冻结一份基线 |
| 配置项标识 | 文档、源代码、测试材料统一编号；版本号规则一致 |

## 7.3 评审、测试、监造、监制、验收

### 评审节点

| 节点 | 时机 | 出口准则 | 评审形式 |
|---|---|---|---|
| 功能与界面设计评审 | ≤10 工作日 | 功能边界、界面原型通过 | 内部 + 甲方 |
| 总体与概要设计评审 | ≤15 工作日 | 架构、技术选型、数据库设计通过 | 内部 + 甲方 |
| 详细设计评审 | 编码前 | 五大模块详细设计文档通过 | 内部 |
| 单元测试评审 | ≤60 工作日 | 单元测试覆盖与通过率达标，注释率 ≥30% | 内部 |
| 合格性测试评审 | ≤90 工作日 / 2026-08-31 前 | 合格性测试报告通过 | 内部 + 甲方 |

评审问题统一记录，按"提出—分析—修改—复核—关闭"流程处理。

### 测试入 / 出口准则

| 阶段 | 入口准则 | 出口准则 |
|---|---|---|
| 单元测试 | 模块代码自检通过、静态检查无高级别问题 | 主流程用例 100% 通过、注释率 ≥30% |
| 集成测试 | 单元测试通过、接口对接条件具备 | 接口用例 100% 通过、关键缺陷为 0 |
| 合格性测试 | 集成测试与试运行通过 | 测试用例 100% 通过，无遗留高级别缺陷 |

### 监制与验收

由甲方按项目计划开展，乙方按要求提供材料、提供测试环境，并参加联合评审。

## 7.4 文件与记录归档

| 归档项 | 介质 | 责任人 |
|---|---|---|
| 需求文档 | 电子 + 纸质 | 项目经理 |
| 设计文档 | 电子 + 纸质 | 架构师 |
| 测试用例与记录 | 电子 | 测试组长 |
| 缺陷库 | 电子 | 测试组长 |
| 变更记录 | 电子 | 配置管理员 |
| 源代码 | 电子（Git 仓库） | 配置管理员 |
| 评审记录 | 电子 + 纸质 | 项目经理 |

归档材料保存期不少于质保周期；甲方有权对上述记录、文件进行检查。

## 7.5 技术状态管理

按 GJB 438C-2021 控制技术状态，关键节点冻结基线：

| 基线 | 冻结时机 | 内容 |
|---|---|---|
| 功能基线 | 功能与界面设计评审通过 | 功能定义、界面原型 |
| 分配基线 | 总体与概要设计评审通过 | 总体设计、概要设计、数据库设计 |
| 产品基线 | 合格性测试评审通过 | 源代码、可执行程序、用户手册 |

版本号规则：`主版本.次版本.修订号-补丁号`，主版本对应基线，修订号对应缺陷修复。

## 7.6 关重件控制

本产品不是关重件。

## 7.7 质保周期

| 项 | 内容 |
|---|---|
| 质保期 | 5 年 |
| 参与活动 | 系统联调、所检、外场、用户验收 |
| 响应方式 | 按甲方要求派人到场，共同解决期间出现的技术问题 |

## 7.8 随机文件齐套性

| 序号 | 文件 | 形式 | 数量 |
|---|---|---|---|
| 1 | 软件设计说明 | 电子 | 一套 |
| 2 | 软件用户手册 | 电子 | 一套 |
| 3 | 软件安装包 | 电子 | 一套 |
| 4 | 软件源代码 | 电子 | 一套 |

文档格式按甲方提供的模板执行；源代码注释行 ≥ 总代码行 30%。
