# 评分方法论

投标书一切以"得分"为核心。本方法论指导 analyst 拆点、writer 应答、expert 打分对齐同一套评分逻辑。

## 一、评分点拆解原则

把招标"评标办法"拆成**可应答的最小评分点**：

1. **可应答粒度**：一个评分点对应一段可独立应答的内容。"技术方案（40 分）"太粗，要拆成总体架构、安全设计、容灾、可扩展性等子点。
2. **分值守恒**：所有评分点分值合计 = 招标评分总分。
3. **客观分 vs 主观分**：
   - 客观分：有明确量化标准（如"提供 X 个同类业绩，每个 2 分"）→ 达标即满分，应答只需举证。
   - 主观分：评委裁量（如"方案合理性，优 8-10 / 良 5-7 / 中 1-4"）→ 应答靠论证质量分档。
4. **资格/符合性项**：即使 0 分，也作为否决项纳入台账。

## 二、应答与得分的对应

| 评分点类型 | 得分关键 | 应答策略 |
|-----------|---------|---------|
| 客观-举证类 | 证据齐全 | 直接给清单/证明/参数，逐条对应 |
| 客观-参数类 | 满足/优于指标 | 偏离表正偏离，突出冗余 |
| 主观-方案类 | 论证深度+差异化 | 结论先行→量化论证→图表→案例佐证 |
| 主观-管理类 | 制度完整+可落地 | 流程图+制度条款+量化指标 |

## 三、主观分分档标准（expert 打分用）

| 档次 | 占满分 | 特征 |
|------|-------|------|
| 优 | 85%-100% | 论证充分、有量化、有图表、有差异化亮点、贴合本项目 |
| 良 | 65%-84% | 论证较完整、部分量化、基本贴合 |
| 中 | 40%-64% | 有应答但偏泛、缺量化、通用模板感 |
| 差 | 0%-39% | 仅口号/缺失/跑题 |

## 四、提分优先级（重写闭环用）

重写时优先攻"投入产出比最高"的失分项：
1. 缺失应答的评分点（从 0 到有，提分最大）
2. 主观分从"中"提到"良/优"（量化 + 图表 + 案例）
3. 客观分举证补全（业绩/资质/参数证明）

## 五、废标优先于评分

任何评分优化都不得以触碰废标红线为代价。expert 先做废标判定，命中即 0 分。
