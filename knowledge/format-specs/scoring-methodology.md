# 技术评分方法论

本系统只编写技术方案，评分只针对**评分表-技术部分**。本方法论指导 analyst 拆技术点、writer 应答技术点、expert 按技术细则打分对齐同一套逻辑。

> 不处理价格分、商务分。台账与评分仅覆盖技术部分。

## 一、技术评分点拆解原则

把"评分表-技术部分"拆成**可应答的最小技术评分点**：

1. **可应答粒度**：一个技术评分点对应一段可独立应答的技术内容。"技术方案（40 分）"要拆成总体架构、详细设计、安全、性能、实施等子点。
2. **分值守恒**：所有技术评分点分值合计 = 评分表技术部分总分。
3. **客观技术分 vs 主观技术分**：
   - 客观技术分：有明确量化标准（如"提供 X 技术指标达标，得 N 分"）→ 达标即满分，应答举证。
   - 主观技术分：评委裁量（如"技术方案合理性，优 8-10 / 良 5-7 / 中 1-4"）→ 应答靠技术论证质量分档。
4. **实质性技术项**：实质性要求-技术产品部分的★项作为否决项纳入废标清单（非评分台账）。

## 二、技术应答与得分对应

| 技术评分点类型 | 得分关键 | 应答策略 |
|--------------|---------|---------|
| 客观-举证类 | 技术证据齐全 | 给技术指标/检测报告/设计值，逐条对应 |
| 客观-参数类 | 满足/优于指标 | 技术偏离表正偏离，突出冗余 |
| 主观-方案类 | 论证深度+技术先进性 | 结论先行→量化论证→图表→案例佐证 |
| 主观-实施类 | 可行性+技术细节 | 技术流程+关键控制点+量化 |

## 三、主观技术分分档（expert 打分用）

| 档次 | 占满分 | 特征 |
|------|-------|------|
| 优 | 85%-100% | 技术论证充分、有量化、有图表、有先进性、贴合本项目技术要求 |
| 良 | 65%-84% | 技术论证较完整、部分量化、基本贴合 |
| 中 | 40%-64% | 有应答但偏泛、缺技术深度、通用模板感 |
| 差 | 0%-39% | 仅口号/缺失/跑题 |

## 四、提分优先级（重写闭环用）

1. 缺失的技术评分点（从 0 到有，提分最大）
2. 主观技术分从"中"提到"良/优"（补量化 + 图表 + 技术细节）
3. 客观技术分举证补全（技术指标/检测/设计值）

## 五、实质性技术要求优先于评分

任何技术评分优化都不得以触碰实质性技术要求红线为代价。expert 先做实质性技术否决判定，命中即 0 分。
