# 技术方案章节格式规范

指导 analyst 生成技术方案大纲、writer 撰写章节、bid-agent 合成最终技术方案的统一格式。

> 本系统只产出**技术方案**（output/technical-proposal.md），不含投标函、商务、报价、资质业绩等章节（除非招标要求在技术方案中附技术性承诺/技术文件）。

## 技术方案典型组成（按技术方案类型裁剪）

```
封面（项目名称 / 技术方案 / 日期）
目录
1. 项目技术理解与需求分析
2. 总体技术方案与技术架构
3. 详细技术设计
   - 软件：**系统总体技术架构（逻辑分层/数据主流向/组件图/部署图）+ 逐功能模块设计（功能模块描述/操作步骤/类与算法设计/用例描述/界面设计）+ 数据库设计 + 接口设计**
   - 硬件：电路/结构/选型/关键指标
   - 集成：集成架构/选型清单/接口对接
   - 服务：服务技术体系/技术流程/支撑平台
4. 非功能/质量技术设计（性能/安全/可靠性/可用性）
5. 实施技术方案（部署/迁移/割接/测试/联调，技术侧）
6. 技术保障措施
7. 技术偏离表（对实质性要求-技术产品部分/技术任务书逐条应答）
```

## 章节写作规范

1. **标题层级**：`#` 项目名 → `##` 一级章 → `###` 二级节 → `####` 三级。与技术方案大纲编号一致。
2. **技术评分点对照**：每章开头放本章承接的技术评分点对照表（推荐）。
3. **应答标记**：每个技术评分点应答段以 `【应答技术评分点 X.Y：名称（分值）】` 开头。
4. **表格优先**：技术参数、指标、清单、偏离一律用表格。
5. **图表**：架构/拓扑/流程/功能模块用 Mermaid。
6. **占位规范**：缺失的真实技术信息用 `【待提供：xxx】`，禁止臆造技术参数。
7. **范围红线**：不写商务/报价/资质业绩；遇到此类内容不纳入技术方案。

## 软件开发类强制完整性

软件类技术方案的"详细技术设计"必须完整包含（缺项视为设计不完整，reviewer 必查）：

**A. 系统总体技术架构（四项必含）：** 逻辑分层、数据主流向、组件图、部署图（四图均出 Mermaid，见 diagram-spec.md）。

**B. 每个功能模块的设计（逐模块，五要素必含）：** 对技术任务书要求的每一个功能模块，统一套用下方模板：

```markdown
### 功能模块 N：{模块名}
- **功能模块描述**：目标/职责/与其他模块关系/对应技术任务书需求点
- **操作步骤**：分步操作流程（可配流程图/时序图）
- **类与算法设计**：核心类（类图/类职责）+ 关键算法（流程/伪代码/复杂度）
- **用例描述**：参与者 / 前置条件 / 主流程 / 备选流程 / 后置条件
- **界面设计**：关键界面布局与交互（界面草图/线框 + 字段控件说明）
```

**C. 数据库设计 + 接口设计：** E-R/表结构/索引/容量；接口清单/协议/请求响应/鉴权/错误码。

## 合成规范（compose 阶段）

- 按 `analysis/outline.md`（技术方案大纲）顺序拼接 `sections/*.md`
- 自动生成目录
- 嵌入 `architecture/diagrams/` 图（Mermaid 代码块或渲染图引用）
- **附"技术部分评分应答索引表"**：取 `analysis/scoring-checklist.md` 终态（应答位置/状态已回填），作为技术方案附表置于正文之后，方便评委按图索骥定位每个技术评分点的应答位置
- **附"技术偏离表"**：取 `analysis/deviation-table.md` 终态（已逐条应答），作为技术方案附表（硬件/集成类必附）
- 输出到 `output/technical-proposal.md`
- 保留所有 `【待提供：...】` 占位并在文末汇总待补技术清单
