# 投标书章节格式规范

指导 analyst 生成大纲、writer 撰写章节、bid-agent 合成最终稿的统一格式。

## 投标文件典型组成

按招标"投标文件格式/组成"为准，常见顺序：

```
封面（项目名称 / 投标人 / 日期）
目录
1. 投标函及投标函附录
2. 法定代表人身份证明 / 授权委托书
3. 投标保证金证明
4. 技术方案（核心评分章节）
   - 总体技术架构
   - 详细技术设计
   - 安全 / 容灾 / 可扩展性
5. 服务方案 / 供货方案 / 施工组织设计（按标书类型）
6. 项目管理与实施（组织、进度、风险）
7. 团队与人员配置
8. 商务部分（报价、商务条款响应）
9. 资质证明与类似项目业绩
10. 技术偏离表 / 商务偏离表
11. 其他（承诺函、售后等）
```

## 章节写作规范

1. **标题层级**：`#` 项目名 → `##` 一级章 → `###` 二级节 → `####` 三级。与大纲编号一致。
2. **评分点对照**：技术/服务/施工类章节开头放本章承接的评分点对照表（可选但推荐）。
3. **应答标记**：每个评分点应答段以 `【应答评分点 X.Y：名称（分值）】` 开头。
4. **表格优先**：参数、指标、清单、SLA、偏离一律用表格。
5. **图表**：架构/拓扑/流程用 Mermaid；进度用甘特图。
6. **占位规范**：缺失的真实信息用 `【待提供：xxx】`，便于作者补全，禁止臆造。
7. **页眉/页脚信息**：合成时由 bid-agent 从 bid.md 注入（项目名、投标人、编号）。

## 合成规范（compose 阶段）

- 按 `analysis/outline.md` 顺序拼接 `sections/*.md`
- 自动生成目录
- 嵌入 `architecture/diagrams/` 图（Mermaid 代码块或渲染图引用）
- 输出到 `output/bid-document.md`
- 保留所有 `【待提供：...】` 占位并在文末汇总待补清单
