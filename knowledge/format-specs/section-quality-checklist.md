# 章节质量验收清单

**定位：** writer 自检 + reviewer 审查的诊断工具集。最高原则：**应答即得分，得分即合格。**

---

## 最高原则

> **章节质量标准只有一个：能不能拿到对应评分点的分。**

下面所有规则是辅助判断的工具，不是终审。最终是否归档由作者判断。

---

## 一、应答完整性（最高优先级）

- [ ] 本章承接的每个评分点都有 `【应答评分点 X.Y】` 标记的实质应答
- [ ] 应答内容对应招标"应答要求"，不偏题
- [ ] 客观分应答有举证（清单/证明/参数）
- [ ] 主观分应答有论证（结论→量化→图表→案例）
- [ ] 无遗漏评分点（逐条对照 scoring-checklist.md）

## 二、废标规避

- [ ] ★项无负偏离
- [ ] 无低于硬性门槛的参数
- [ ] 格式/签章/承诺类要求已满足或显式占位
- [ ] 无任何触碰 disqualification.md 的表述

## 三、可核验性（反空话）

- [ ] 用指标/数据/表格/流程/清单，而非形容词
- [ ] 承诺有考核方式（如 SLA 有统计口径）
- [ ] 无空喊口号："高度重视""精益求精""一流""国内领先"等无信息量表述
- [ ] 参数/业绩/资质真实或占位标注 `【待提供：...】`，无臆造

## 四、AI 味检测（详见 anti-ai.md）

- [ ] 疲劳词未超阈值（"充分""有效""不断""持续""全面"等）
- [ ] 无连续同句式堆砌
- [ ] 无"综上所述""总而言之"类议论文腔（除非小结需要）
- [ ] 句式克制，公文风，术语准确

## 五、类型专属产物

- [ ] 服务类：SLA 指标表（含考核方式）、团队组织架构、运维流程
- [ ] 采购类：配置清单、逐条技术偏离表应答、兼容性说明
- [ ] 工程类：进度甘特图、质量保障措施、安全文明施工

## 六、一致性

- [ ] 数据/术语/承诺与其他章节、与 architecture/ 一致
- [ ] 引用检索素材已去敏、已标注来源

## 七、技术质量

- [ ] 表格优先于冗长段落
- [ ] 章节结构清晰（标题层级规范）
- [ ] 图表可解析（Mermaid 语法正确）

---

## 验收报告格式

```markdown
## 章节验收报告：第 X 章

### 总评
✅ 通过 / ⚠️ 有条件通过 / ❌ 需修改

### 应答覆盖
| 评分点 | 应答 | 质量 |
|--------|------|------|
| 3.1 | ✅ | 优 |
| 3.2 | ❌ 缺失 | — |

### 未通过项
| 检查项 | 结果 | 证据 | 整改 |
|--------|------|------|------|
```
