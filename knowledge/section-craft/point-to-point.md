# 技法：点对点技术应答

技术方案得分的第一原则——**技术评委按技术评分点找应答，找不到就不给分。** 让每个技术评分点都"一眼可见地被应答"。

## 标准句式

```markdown
【应答技术评分点 3.2：安全设计（6分）】
结论：本项目按等保三级设计，划分四个安全域并部署纵深防御。
论证：① 安全域划分（见拓扑图）……② 边界 WAF+IPS……③ 数据加密……
佐证：参考某省政务云项目（已去敏），同等技术方案通过等保三级测评。
```

四段式：**标记 → 结论 → 论证 → 佐证**。

## 要点

1. **标记前置**：`【应答技术评分点 X.Y：名称（分值）】` 让技术评委秒定位。
2. **结论先行**：首句给明确结论/承诺，评委扫读即得要点。
3. **量化论证**：用指标、表格、图、清单，不堆形容词。
4. **案例佐证**：引用 retriever 提供的去敏案例增强可信度。
5. **逐点不漏**：本章承接的技术评分点逐一应答，宁可重复不可遗漏。

## 反面（失分）

- 把多个技术评分点揉进一段，评委找不到 → 漏给分
- 只有口号无量化 → 主观分掉到"中"档
- 应答跑题（答非所问）→ 0 分
