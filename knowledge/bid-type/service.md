# 服务类标书撰写要点

适用：运维服务、技术支持、咨询、外包、集成服务等以"持续服务能力"为核心的项目。

## 核心得分逻辑

评委关注：**你能不能稳定、可量化地把服务做好。** 一切应答围绕"可承诺、可考核、可追溯"。

## 必备章节与侧重

### 1. 服务方案总述
- 服务范围、服务模式（驻场/远程/混合）、服务时间（5×8 / 7×24）
- 服务理念落到机制，不喊口号

### 2. SLA 服务等级协议（高分核心）
量化指标表 + 考核方式 + 未达标罚则：
| 指标 | 承诺值 | 统计口径 | 未达罚则 |
|------|-------|---------|---------|
| 系统可用率 | ≥99.9% | 月度 | 扣服务费 X% |
| 一般故障响应 | ≤30 分钟 | 工单时间戳 | ... |
| 严重故障响应 | ≤15 分钟 | ... | ... |
| 故障恢复 | ≤2 小时 | ... | ... |

### 3. 团队架构与人员配置
- 组织架构图（项目经理→技术组/服务台/质量组）
- 关键岗位人员资质、证书、同类项目经验（可引用业绩）
- 人员到岗承诺与替补机制

### 4. 运维管理制度
- 制度清单：变更管理、配置管理、问题管理、知识库管理、值班制度
- ITIL/ISO20000 对标（如招标要求）

### 5. 服务响应流程
- 分级响应流程图（sequenceDiagram）
- 工单全生命周期：受理→定级→派单→处理→恢复→回访→复盘
- 升级机制（超时自动升级）

### 6. 应急预案与持续改进
- 重大故障应急预案、容灾切换流程
- 月度/季度服务报告与持续改进 PDCA

## 常见失分点

- SLA 只写"快速响应"无量化 → 主观分掉档
- 团队无资质举证 → 客观分丢分
- 制度堆名词无流程 → 评委认为不可落地
- 无未达标罚则 → 承诺可信度低

## 常见废标点

- 关键岗位资质/证书不满足★要求
- 服务时间承诺低于招标硬性要求
- 缺法定授权/承诺函
