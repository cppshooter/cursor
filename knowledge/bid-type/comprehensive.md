# 综合类标书撰写要点

适用：同时包含服务、采购、工程多重要素的复合型项目（如"软硬件采购+集成实施+运维服务"一体化项目）。

## 核心得分逻辑

按招标评分细则的实际构成，**分模块套用对应类型要点**，并强调各模块的协同与一体化交付能力。

## 撰写策略

1. **拆模块对齐评分**：先按 scoring-checklist 看分值分布在采购/工程/服务的比重，资源投入向高分模块倾斜。
2. **各模块套用对应要点：**
   - 采购模块 → 见 `bid-type/procurement`（配置清单 + 技术偏离表）
   - 工程/实施模块 → 见 `bid-type/engineering`（施工组织 + 进度甘特图）
   - 服务/运维模块 → 见 `bid-type/service`（SLA + 团队 + 流程）
3. **强调一体化**：突出"供货-实施-运维"全生命周期由同一团队闭环交付，减少接口风险。
4. **总体架构统领**：用 architect 的总体技术架构把各模块串成一个方案，避免拼盘感。

## 必备章节（典型）

```
1. 项目总体理解与一体化方案
2. 总体技术架构（architect 产出）
3. 产品采购与配置（采购要点 + 偏离表）
4. 集成实施方案（工程要点 + 进度甘特图）
5. 运维服务方案（服务要点 + SLA）
6. 项目管理与团队
7. 商务与资质业绩
8. 技术偏离表 + 商务偏离表
```

## 常见失分点

- 各模块割裂，缺一体化论证 → 主观分掉档
- 高分模块投入不足，低分模块过度展开 → 资源错配
- 跨模块数据/承诺不一致（如工期与运维起算时间矛盾）→ reviewer 必查

## 常见废标点

- 任一模块触碰其对应类型的废标红线（★项负偏离 / 关键资质缺失 / 工期超限等）
- 遗漏招标要求的某一模块响应
