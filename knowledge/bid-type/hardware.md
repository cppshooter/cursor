# 硬件开发类技术方案撰写要点

适用：硬件产品/设备的研发设计、定制开发、生产类项目的**技术方案**编写。

> 编写依据只取招标文件的：**评分表-技术部分**、**实质性要求-技术产品部分**、**技术任务书 / 采购需求技术部分**。不涉及商务、报价、资质业绩等内容。

## 核心得分逻辑

技术评委关注：**硬件设计是否满足甚至优于技术指标、可靠性如何、是否可量产可验证。** 技术参数偏离表是核心。

## 强制完整性（缺项即失分/视为设计不完整）

硬件类技术方案**必须**完整包含以下内容，逐项不可缺：

1. **需求与技术指标逐项分解**（每条招标技术指标对应到设计指标）
2. **硬件总体设计** —— 必须包含：**功能框图、技术路线、关键器件/芯片选型与理由**
3. **详细设计** —— 必须包含：**电路设计（原理/关键电路）、结构设计、热设计、接口与信号/电源完整性**
4. **关键技术指标响应** —— 形成**技术参数偏离表**（逐条设计值/实测值，**★项无偏离/正偏离，无空项**）
5. **可靠性与合规** —— 必须包含：**可靠性设计（冗余/降额/MTBF）、EMC/EMI、环境适应性、强制认证/标准符合性**
6. **生产工艺与测试验证** —— 工艺方案 + 关键工序质量控制 + 测试验证（功能/性能/老化/环境）

## 必备章节与侧重

### 1. 需求理解与技术指标分解
- 对技术任务书/采购需求技术部分的指标逐项分解
- 关键技术指标与难点

### 2. 硬件总体设计方案
- 总体方案、功能框图、技术路线
- 关键器件/芯片选型与理由

### 3. 详细设计
- 电路设计（原理/关键电路）、结构设计、热设计
- 接口设计、信号完整性/电源完整性

### 4. 关键技术指标响应
- 逐条响应技术参数（实测/设计值），形成**技术参数偏离表**
- ★项必须无偏离/正偏离

### 5. 可靠性与合规
- 可靠性设计（冗余/降额/MTBF）、EMC/EMI、环境适应性
- 强制认证与标准符合性（3C/能效/行业标准）

### 6. 生产工艺与测试验证（技术侧）
- 工艺方案、关键工序质量控制
- 测试验证方案（功能/性能/老化/环境试验）

## 常见失分点

- 技术指标响应笼统"满足"，无设计值/实测值
- 缺可靠性/EMC 设计论证
- 偏离表有空项或★项负偏离
- 选型无理由、与指标不匹配

## 实质性要求（技术产品部分）红线

- 实质性★技术指标负偏离 → 否决
- 强制认证/标准未满足 → 否决
- 必须提供的技术参数/检测报告承诺缺失 → 否决

## 撰写完整性自检清单（每章生成后逐项核对）

- [ ] 需求与技术指标逐项分解，每条指标可追溯
- [ ] 硬件总体设计：功能框图 + 技术路线 + 关键器件选型理由
- [ ] 详细设计：电路 + 结构 + 热设计 + 接口/信号完整性/电源完整性
- [ ] 技术参数偏离表逐条填写（设计值/实测值），★项无负偏离、无空项
- [ ] 可靠性与合规：可靠性设计 + EMC/EMI + 环境适应性 + 强制认证
- [ ] 生产工艺与测试验证齐备
- [ ] 强制认证/标准已响应，必须提供的检测报告承诺已给出
