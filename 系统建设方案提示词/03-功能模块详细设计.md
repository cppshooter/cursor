# 03 功能模块详细设计章节总提示词

请你作为系统详细设计文档编写专家，根据《系统需求.md》和《大纲.md》编写“3. 功能模块详细设计”的章节总述，并组织各模块的编号和编写规则。

## 通用编写约束

- 严格依据《系统需求.md》和《大纲.md》编写，不新增需求未提出的大模块、外部系统、用户角色或业务流程。
- 可以在既有功能要求内部拆分功能细节，但拆分必须服务于原始要求的解释、落地和验收。
- 语言采用正式、客观、工程化的系统建设方案表述。
- 涉及技术方案时优先体现成熟技术、模块化、高聚合低耦合、输入合法性检查、异常处理和误操作防护。
- 操作系统约束统一写为：银河麒麟 V10。
- 接口边界统一限定为：控制指令、状态信息、指令执行结果、任务结果报送、技术状态报送。
- 输出时禁止使用占位性措辞或空泛留白。


## 章节边界

本章只包含以下五个功能模块，不新增其他大模块：

1. 任务管理模块：任务接收、结果报送。
2. 数据处理模块：自动处理、手动处理。
3. 硬件交互模块：电源开机控制、状态信息接收、返回结果文件保存、离线文件处理、交互过程入库、结果入库。
4. 结果评估模块：硬件结果接收、接收效果评判、时序列表生成、时序列表手动调整、算法显示结果。
5. 系统管理模块：状态监控、策略管理、信息管理。

## 每个模块必须采用的统一详细设计结构

- 功能模块描述：说明模块职责、输入、输出、边界和约束。
- 操作步骤：按用户或系统触发顺序描述流程，并加入简单明了的 PlantUML 时序图。
- 类 / 算法设计：使用 Java 代码描述架构类、接口、核心算法或关键对象关系，不写完整工程代码。
- 用例描述：描述参与者、用例和主要交互，并使用 PlantUML 绘制简单明了的用例图。
- 界面设计：使用 HTML 代码描述界面结构，体现输入、操作按钮、状态展示、结果展示和异常提示。

## 输出格式要求

- 先写第 3 章总体说明，再列出 3.1 至 3.5 的模块目录。
- 不在总述文件中展开完整模块设计，完整内容由模块级提示词生成。
- 明确各模块之间通过任务编号、指令编号、硬件状态、处理结果、策略参数、数据库记录等数据关联。
