# 通用反 AI 味 + 公文文风规则

## 一、空话套话黑名单（删除或替换为可核验内容）

| 套话 | 问题 | 替代 |
|------|------|------|
| 我公司高度重视 | 无信息量 | 删，直接给措施 |
| 精益求精 / 追求卓越 / 匠心 | 口号 | 删，给质量指标 |
| 国内一流 / 行业领先 | 无证据自夸 | 删或给可举证排名/资质 |
| 充分发挥 / 切实保障 / 全力以赴 | 空泛 | 删，给具体机制 |
| 不断完善 / 持续优化 | 无操作性 | 给 PDCA 周期与指标 |
| 强大的 / 丰富的 / 雄厚的 | 主观形容 | 给数字（X人/X个业绩） |
| 以人为本 / 服务至上 | 标语 | 删，给 SLA |

## 二、AI 疲劳词阈值（单章）

| 词 | 阈值 | 说明 |
|----|------|------|
| 充分 | ≤3 | 多为填充，删之意思不变 |
| 有效 | ≤3 | 用具体效果替代 |
| 不断 / 持续 | ≤2 | 给周期/频次替代 |
| 全面 / 全方位 | ≤2 | 列具体范围替代 |
| 进一步 | ≤2 | 删 |
| 相关 | ≤3 | 指明具体对象 |
| 通过……，实现…… | ≤4 | 句式套路，变换表达 |

## 三、句式规则

1. **结论先行**：每个应答/段落首句给结论或承诺，不绕弯子。
2. **避免连续同句式**：禁止连续 4 句以"通过/为了/在……方面"开头。
3. **慎用议论文腔**：除小结外不用"综上所述""总而言之"开篇堆砌。
4. **表格优先**：能列表的不写长段落（参数、指标、清单、流程）。
5. **术语准确统一**：专业术语前后一致，缩写首次出现给全称。

## 四、内容真实性（红线）

- 不臆造业绩、资质、参数、人员 —— 缺失用 `【待提供：...】` 占位
- 业绩/资质须真实可举证（虚假应答可致废标乃至处罚）
- 引用案例标注"已去敏"，不泄露真实客户敏感信息

## 五、格式规范

- 用规范的标题层级，与大纲编号一致
- 不在正文出现"作为 AI""以下是""希望对您有帮助"等助手腔
- 不出现 Markdown 之外的奇怪标记；图用 Mermaid 代码块
