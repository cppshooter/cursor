# knowledge-retrieval skill

## 流程概览

```text
Step 1: 解析检索需求（本章技术评分点 + 技术素材类型）
Step 2: 案例库匹配（类型/规模/技术栈相关度）
Step 3: 提取可引用片段与技术参数
Step 4: 去敏处理
Step 5: 结构化产出素材清单
```

## Step 1: 解析检索需求

读 `.agent/task/retrieval-order.md`：本章号、待应答的技术评分点、需要的**技术素材**类型（技术案例描述 / 技术参数基准 / 技术方案片段 / 技术能力指标基准）。仅检索技术素材，不检索商务/资质业绩。

## Step 2: 案例库匹配

读 `.claude/knowledge/case-library.md`，用 Grep 按关键词检索。相关度排序维度：
1. **同类型**（软件开发/硬件开发/系统集成/服务）优先
2. **同规模**（预算/体量相近）
3. **同技术栈/同行业**
4. **技术评分点契合**（案例覆盖了本章要应答的技术点）

## Step 3: 提取可引用片段

每个命中案例提取：
- 成功案例描述（项目背景、解决方案要点、成效数据）
- 技术参数表（可作为响应/偏离应答的参照）
- 关键指标（SLA、工期、质量合格率等基准）

## Step 4: 去敏

- 真实客户名/项目全称 → 泛化（"某省级政务云项目""某大型制造企业"）
- 涉密/合同金额/个人信息 → 移除或区间化
- 标注"已去敏，可引用"

## Step 5: 结构化产出 → `.agent/task/retrieval-result.md`

```markdown
# 本章参考素材（ch-{N}）

## 技术评分点 3.1 总体架构
### 素材1（相关度：高）
- 来源：案例库 / 某省级政务云（已去敏）
- 可引用片段：采用双活数据中心架构，RPO≤5分钟，RTO≤30分钟……
- 技术参数：双AZ、容器化、等保三级
- 适配建议：可作为本项目容灾设计的成功佐证

### 素材2（相关度：中）
- ...

## 技术评分点 3.2 ...
- 未命中：案例库无直接匹配，建议作者补充历史技术方案 / 采用通用行业技术基准（标注为基准）
```

## 验证（Definition of Done）

- [ ] order 中每个待检索技术评分点都有素材或明确"未命中"
- [ ] 每条素材含相关度/片段/参数/来源/适配建议
- [ ] 全部已去敏
- [ ] 无臆造案例、无主观撰写正文
