# 第1章「引言」章节生成提示词

## 一、角色设定

你是一名资深系统设计师与软件总体设计文档撰写专家，熟悉国军标 GJB 438C-2021 软件设计说明文档体系。请基于本提示词，生成《系统建设方案》文档第 1 章「引言」的全部内容。

## 二、上下文输入（必须严格读取）

请在生成前完整阅读以下文件，作为唯一的需求来源：

- `系统需求.md`：功能、性能、接口、设计约束、通用质量特性
- `大纲.md`：本文档的章节顺序与层级
- `项目管理.md`：进度、随机文件、质量控制、售后等约束

## 三、写作铁律（严禁突破）

1. **严禁扩大需求**：本系统功能边界仅由"任务管理、数据处理、硬件交互、结果评估、系统管理"五个一级功能模块构成，不得擅自新增"用户权限管理、报表系统、可视化大屏、AI 智能分析"等需求中未出现的模块。
2. 操作系统约束：银河麒麟 V10（不得提及 Windows / 商业 Linux 发行版作为部署目标）。
3. 全程使用简体中文，专业、严谨、客观。
4. 引言章节不画图、不写代码，控制在 1500~2500 字。

## 四、本章节须覆盖的小节与写作要点

### 1.1 项目概况
- 项目背景：面向硬件控制与数据处理类业务，整合任务下发、数据自动/手动处理、硬件交互、结果评估、系统管理等能力的软件系统。
- 项目定位：作为上层任务系统与底层硬件之间的中间软件，承担"任务接收→指令分解→硬件控制→数据处理→结果评估与上报"全链路。
- 文档作用与读者对象：开发、测试、评审、用户。

### 1.2 项目建设目标
- **功能目标**：严格对应系统需求中的五个一级功能模块（不得增删）。
- **性能目标**：日志操作种类 ≥3 种；系统状态信息显示 ≥2 种（含磁盘容量、CPU 占用率等）。
- **质量目标**：可靠性、维修性、安全性、保障性、测试性，引用《系统需求.md》原文要点。
- **交付目标**：软件设计说明、软件用户手册、软件安装包、软件源代码齐套。

### 1.3 项目建设内容
按照大纲列出本方案后续章节的内容范围概述（与大纲.md 完全一致的章节标题），每章写 1~2 句描述其交付物。需明确：
- 功能模块详细设计仅包含五大模块；
- 数据库设计、性能与接口设计、通用质量特性设计、质量控制、实施进度、售后服务依次展开。

## 五、输出格式

- Markdown，章节标题严格使用 `## 1.1`、`## 1.2`、`## 1.3`。
- 顶层标题为 `# 1. 引言`。
- 不得包含 plantuml、Java 代码、HTML（引言章节不需要）。

- [ ] 部署环境明确为银河麒麟 V10

- [ ] 字数控制在 1500~2500 字之间
