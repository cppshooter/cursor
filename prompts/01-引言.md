# 第1章「引言」章节生成提示词

## 一、上下文输入（必须随本提示词一起提供）

- `系统需求.md`
- `项目管理.md`
- `_共享_写作规范.md`（附录 A，强制遵守）

## 二、章节定位

引言为全文起手，**仅作背景与范围交代**，不展开论证。**目标页数 ≤2 页**（A4 小四 1.5 倍行距），约 600~900 字。本章不画图、不写代码、不写界面。

## 三、本章三个小节的写作要点

### 1.1 项目概况
- 一段话说明：本系统是部署于银河麒麟 V10 上位机的软件，承接上层任务软件下发的任务，控制电源等受控硬件并完成数据处理、结果评估与上报。
- 注明文档对象：开发、测试、评审与操作员。
- 字数：120~200 字。

### 1.2 项目建设目标
- 用一段或一个简表说明 5 类目标：功能（五大模块）、性能（日志 ≥3 种、系统状态 ≥2 种）、接口（控制指令 / 状态信息 / 指令执行结果）、质量（GJB 438C-2021、注释率 ≥30%）、交付（设计说明、用户手册、安装包、源代码）。
- **不展开论证**，写一条算一条。
- 字数：200~300 字（含表）。

### 1.3 项目建设内容
- 用一个表格列出本方案后续 8 个章节（2~9 章）的标题与一句话内容描述。
- 章节顺序与《大纲.md》完全一致。
- **不重复"将在第 X 章详细描述"等冗余引导句**。
- 字数：200~300 字（含表）。

## 四、写作铁律

1. 遵守附录 A 的全部禁用词与禁用句式（赋能、抓手、闭环、打造、综上所述等禁止出现）。
2. 不在引言里讨论架构、不在引言里写技术选型细节、不在引言里展开质量特性。
3. 五大功能模块名称按需求原文：**任务管理 / 数据处理 / 硬件交互 / 结果评估 / 系统管理**。
4. 开发框架统一表述为 "C++17 + Qt 5.15 LTS"，操作系统统一表述为 "银河麒麟 V10"。

## 五、输出格式

- Markdown，顶层 `# 1. 引言`，子标题 `## 1.1`、`## 1.2`、`## 1.3`。
- 仅文字与必要表格，无图、无代码、无 HTML。

## 六、自检

- [ ] 总字数 600~900 字、版面 ≤2 页
- [ ] 未出现禁用词/AI 味套话
- [ ] 三小节内容不互相重复
- [ ] 章节内容描述与《大纲.md》一一对应
