## 三、需求映射（严格不扩大）

来自《系统需求.md》「数据处理」原文：

- （1）自动处理：能够自动控制硬件进行数据处理并记录处理过程；能够生成、读取、导出数据库表；能够基于数据库表比对；能够引导其他功能；能够对异常数据进行告警。
- （2）手动处理：能够对数据进行显示；能够手动控制硬件进行数据处理；能够数据进行拼接；能够自动解析文件名称；能够调用指定算法；能够自动生成 Excel 表；能够把数据保存至文件。

仅拆分为：

- **3.2.1 自动处理子模块**（覆盖 5 条）
- **3.2.2 手动处理子模块**（覆盖 7 条）

不引入"清洗规则引擎、智能分析、AI 异常检测训练"等。

## 四、每个子模块固定五小节

### (1) 功能模块描述
- 表格罗列：输入（数据来源：硬件 / 文件 / 数据库表）、输出（数据库记录 / Excel / 文件 / 告警事件）、依赖（硬件交互模块、数据库、文件系统）。
- 1 段话说明本子模块如何串联输入→处理→输出。

### (2) 操作步骤（用户操作手册风）

**自动处理子模块** 操作步骤建议（任选 1~2 条主流程展开为 10 步内）：

- 入口示例：
  - 主菜单 `工具(T) → 自动处理(A)`，或工具栏【启动自动处理】按钮。
  - 左侧导航树勾选目标硬件后，点击【运行】按钮。
- 表单字段：硬件选择（`QComboBox`）、处理策略选择（`QComboBox`）、目标表（`QComboBox`）、告警阈值（`QSpinBox`）。
- 操作明确：
  - 点击【开始】后，进度条 `.qt-progress` 显示百分比；过程日志写入下方 `.qt-log`。
  - 处理结束后，状态栏提示"完成 N 条记录"，告警计数显示在状态栏右侧。
  - 在【数据库表】Tab 内可选择【生成新表】【读取表】【导出表】【比对两表】。
  - 异常告警弹出 `QDialog`，并在告警列表中追加一行。

**手动处理子模块** 操作步骤建议（任选 1~2 条主流程展开为 10 步内）：

- 入口示例：
  - 主菜单 `工具(T) → 手动处理(M)`，或工具栏【手动处理】按钮。
- 表单字段：文件选择（【浏览】打开 `QFileDialog`）、文件名解析结果展示（只读 `QLineEdit` × N 个字段）、算法选择（`QComboBox`）、拼接源选择（多选 `QListWidget`）。
- 操作明确：
  - 解析文件名：点击【解析】按钮，自动填充元数据字段；解析失败弹窗提示。
  - 数据拼接：勾选多文件 → 点击【拼接】 → 在中央 `QTableView` 中预览。
  - 算法调用：选择算法 → 点击【运行】 → 结果显示于 `QTableView` 与 `.qt-log`。
  - 导出 Excel：点击【导出 Excel】 → 弹 `QFileDialog` 选择保存路径。
  - 保存到文件：点击【另存为】 → 选择路径与格式（`*.csv` / `*.txt` / `*.dat`）。
- 每个步骤都要写"在界面 X 区域执行 Y 操作，观察 Z 反馈"，与界面 HTML 一一对应。
- 紧跟一张 **PlantUML 时序图**（≤5 参与者）。

### (3) 类与算法设计（C++17 + Qt）
- 自动处理建议类：
  - `AutoProcessor : public QObject`（信号 `progressChanged(int)`、`anomalyDetected(Anomaly)`；槽 `start()`、`stop()`）
  - `DbTableService`（基于 `QSqlDatabase`，提供 `createTable / readTable / exportTable`）
  - `TableComparator`（基于主键 + 字段差异比对）
  - `AnomalyDetector`（阈值/统计偏差）
  - `ProcessLogger`（写入 `t_auto_process_log`）
- 手动处理建议类：
  - `ManualProcessor : public QObject`
  - `FileNameParser`（正则/规则 → 元数据 struct）
  - `DataConcatenator`
  - `AlgorithmInvoker`（注册-调用模式）
  - `ExcelExporter`（基于 QXlsx）
  - `FileSaver`
- 必选 1 个核心算法（≤30 行 C++）：**数据库表比对算法**（主键交集 + 字段差异输出）或 **文件名解析算法**（正则 + 命名规则映射）。

### (4) 用例描述（PlantUML 用例图）
- 自动处理用例：自动控制硬件采集、记录处理过程、生成/读取/导出数据库表、表比对、引导其他功能、异常告警。
- 手动处理用例：显示数据、手动控制硬件处理、数据拼接、解析文件名、调用算法、导出 Excel、保存到文件。
- 节点 ≤12 个，可分两张图或合并一张。

### (5) 界面设计（HTML，严格套用附录 B 模板）
- 共同：复制附录 B 骨架，菜单/状态栏保留。
- 自动处理界面 `qt-content` 区建议：
  - 顶部 `.qt-group "运行控制"`：硬件选择、策略选择、目标表、阈值、【开始】【停止】按钮，进度条。
  - 中部 `.qt-tabs`（数据库表 / 比对结果 / 告警列表）
    - 数据库表 Tab：表格 + 【生成新表】【读取】【导出】【比对】按钮组
    - 比对结果 Tab：左右两 `.qt-table` 显示差异
    - 告警 Tab：`.qt-table` 列（时间、级别、内容、操作）
  - 底部 `.qt-log` 处理过程日志
- 手动处理界面 `qt-content` 区建议：
  - 顶部 `.qt-group "文件解析"`：文件路径 + 【浏览】+【解析】，下方解析字段只读展示
  - 中部 左右分栏：
    - 左：算法面板（算法选择、参数表、【运行】、【调用其他算法】）
    - 中央：数据显示 `.qt-table`（可切换为曲线占位区）
    - 右：拼接源列表 `.qt-tree` + 【拼接】按钮
  - 底部按钮区：【导出 Excel】【另存为】【清空】
