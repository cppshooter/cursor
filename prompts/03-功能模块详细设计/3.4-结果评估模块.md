# 第3.4节「结果评估模块」详细设计 提示词

## 二、章节定位与篇幅

第 3 章第 4 个一级模块。**目标页数 6~8 页**。

## 三、需求映射（严格不扩大）

来自《系统需求.md》「结果评估」原文：

> 能够接收硬件返回结果；能够评判结果接收效果；能够生成时序列表；能够在时序列表手动调整内容；能够调用算法显示结果。

拆分为：

- **3.4.1 硬件结果接收与评判子模块**
- **3.4.2 时序列表管理子模块**
- **3.4.3 算法调用与结果展示子模块**

不引入"评估模型训练、智能打分、报告自动撰写"。

## 四、每个子模块固定五小节

### (1) 功能模块描述
- 输入（硬件交互模块结果 / 操作员调整 / 算法库）、输出（评判等级、时序列表、结果可视化）、依赖（硬件交互模块、数据处理模块）。

### (2) 操作步骤（用户操作手册风）

**3.4.1 接收与评判** 操作步骤建议：
- 入口：主菜单 `视图(V) → 结果评估(E)`，或工具栏【接收效果评判】。
- 中央显示"实时接收数据"`.qt-table`（列：序号、时间、长度、校验、状态）。
- 顶部"指标卡" `.qt-group`：完整率、误码率、丢包率、接收评级（A/B/C）。
- 操作：
  - 点击【启动评判】→ 接收来自硬件交互模块的数据，自动滚动表格 + 指标卡刷新。
  - 点击【生成评判报告】→ 弹 `QFileDialog` 保存为文件。
  - 点击【停止】→ 状态栏提示"评判已停止"。

**3.4.2 时序列表管理** 操作步骤建议：
- 入口：主菜单 `视图(V) → 时序列表(T)`，或工具栏【时序列表】。
- 中央 `.qt-table`（可编辑）列：序号、时间、事件、参数、操作。
- 操作：
  - 点击工具栏【生成时序列表】→ 系统根据已有评判结果自动生成行项。
  - 选中一行 → 点击【编辑】或双击 → 进入行编辑（`QLineEdit` / `QComboBox` 内嵌于单元格）。
  - 点击【插入行】【删除行】【上移】【下移】调整顺序。
  - 点击【保存】→ 写入数据库；点击【导出】→ 保存为文件。

**3.4.3 算法调用与结果展示** 操作步骤建议：
- 入口：主菜单 `工具(T) → 算法调用(A)`，或工具栏【运行算法】。
- 表单：算法选择（`QComboBox`，从算法库列出可用项）、参数表（`QTableView`，参数名 + 值）。
- 操作：
  - 点击【运行】→ 调用所选算法 → 结果显示于中央 `.qt-tabs`（表格 / 文本视图 / 简易曲线占位）。
  - 点击【保存结果】→ 写入数据库或导出文件。

每个子模块紧跟一张 **PlantUML 时序图**（≤5 参与者）。

### (3) 类与算法设计（C++17 + Qt）
- 接收/评判：
  - `HardwareResultReceiver : public QObject`（信号 `resultReceived(Result)`）
  - `ReceptionQualityEvaluator`（输入：批数据；输出：`EvaluationResult { rate, errorRate, lossRate, grade }`）
- 时序列表：
  - `TimingListGenerator`（输入：评判结果集合 → 输出 `QList<TimingItem>`）
  - `TimingListEditor : public QAbstractTableModel`
  - `TimingListRepository`（Qt SQL）
- 算法调用：
  - `IAlgorithm`（抽象，`virtual QVariant run(const QVariantMap& params) = 0;`）
  - `AlgorithmRegistry`（注册-查找）
  - `ResultRenderer`（向 `QTableView` / `QTextEdit` / 曲线占位渲染）
- 必选 1 个核心算法（≤30 行 C++）：**接收效果评判算法**（完整率/误码率/丢包率加权打分 → 评级）或 **时序列表生成算法**（按时间戳排序 + 关键事件抽取）。

### (4) 用例描述（PlantUML 用例图）
- 参与者：操作员、硬件交互模块、算法库。
- 用例：接收硬件返回结果、评判结果接收效果、生成时序列表、手动调整时序列表、调用算法显示结果。
- 节点 ≤10。

### (5) 界面设计（HTML，严格套用附录 B 模板）
- 整体在主窗口下用 `.qt-tabs` 切换 3 个子界面，或分别给出 3 段 HTML（统一骨架）。
- 工具栏建议按钮：【启动评判】【停止】【生成评判报告】【生成时序列表】【插入行】【删除行】【上移】【下移】【运行算法】。
- 中央工作区：
  - 接收与评判：顶部"指标卡" 4 个（完整率/误码率/丢包率/评级），下方实时接收 `.qt-table`，底部 `.qt-log` 评判日志。
  - 时序列表：中央 `.qt-table`（可编辑），右侧 `.qt-group "字段编辑"`，底部按钮区【保存】【导出】。
  - 算法调用与展示：左侧算法树 `.qt-tree`，中央参数表 + 【运行】按钮，右侧 `.qt-tabs`（表格 / 文本 / 曲线占位）。
- 状态栏：【接收速率】【评级】【数据库连接】`.qt-led`。
