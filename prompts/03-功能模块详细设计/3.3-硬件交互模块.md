## 二、章节定位与篇幅

第 3 章第 3 个一级模块。**目标页数 6~8 页**。

## 三、需求映射（严格不扩大）

来自《系统需求.md》「硬件交互」原文：

> 能够控制电源开机；能够接收电源状态信息；能够对硬件返回的结果保存为文件；能够离线处理文件；能够对硬件交互过程进行入库；能够对结果进行入库。

拆分为：

- **3.3.1 电源控制子模块**（控制电源开机；接收电源状态信息）
- **3.3.2 硬件结果采集与保存子模块**（结果保存为文件；离线处理文件）
- **3.3.3 交互过程与结果入库子模块**（交互过程入库；结果入库）

不引入"固件升级、远程运维、设备拓扑可视化"。

## 四、每个子模块固定五小节

### (1) 功能模块描述
- 表格：输入（控制指令 / 硬件返回 / 文件）、输出（电源状态 / 文件 / 数据库记录）、依赖（QSerialPort / QTcpSocket、数据库）。

### (2) 操作步骤（用户操作手册风）

**3.3.1 电源控制** 操作步骤建议：
- 入口：主菜单 `工具(T) → 电源控制(P)`，或工具栏【电源开机】按钮。
- 表单：通信方式（`QComboBox`：串口 / TCP）、端口号（串口为 `COM*` / `/dev/ttyS*`；TCP 为 IP+Port，`QLineEdit + QSpinBox`）、波特率（`QComboBox`）。
- 操作：
  - 点击【连接】→ 状态栏右侧 `.qt-led` 变绿。
  - 点击【开机】→ 弹出 `QDialog` 二次确认 → 确认后下发指令 → 等待状态回传 → 界面"当前状态"区域刷新（电压、电流、温度等占位）。
  - 状态信息每 1Hz 自动刷新（呼应"接收电源状态信息"），可在界面切换刷新频率。

**3.3.2 硬件结果采集与保存** 操作步骤建议：
- 入口：主菜单 `工具(T) → 结果采集(R)`，或工具栏【在线采集】/【离线处理】按钮。
- 在线采集：
  - 选择"保存路径"（【浏览】→ `QFileDialog`）、文件命名规则（`QComboBox`）。
  - 点击【开始采集】→ 实时数据流写入文件，界面 `.qt-progress` 显示文件大小。
  - 点击【停止】→ 弹窗显示已保存文件路径。
- 离线处理：
  - 切到【离线处理】Tab，点击【加载文件】→ `QFileDialog` 多选。
  - 文件列表 `.qt-table` 显示选中文件；点击【批量处理】→ 进度条 + 处理日志。

**3.3.3 交互过程与结果入库** 操作步骤建议：
- 入口：主菜单 `视图(V) → 交互日志`，或工具栏【入库记录】。
- 中央 `.qt-tabs`：交互日志 / 结果记录。
  - 交互日志 Tab：表格列（时间、方向、指令码、参数、状态）。
  - 结果记录 Tab：表格列（时间、任务 ID、内容摘要、保存路径、状态）。
- 操作：检索（顶部筛选条件 `.qt-input + .qt-combo + .qt-btn`）、清除（`.qt-btn-danger` + 二次确认）、重置筛选（`.qt-btn`）。

每个子模块紧跟一张 **PlantUML 时序图**（≤5 参与者）。

### (3) 类与算法设计（C++17 + Qt）
- 电源控制：
  - `IPowerController`（抽象基类，`virtual bool powerOn() = 0;`）
  - `SerialPowerController : public IPowerController`（基于 `QSerialPort`，信号 `statusReceived(PowerStatus)`）
  - `TcpPowerController : public IPowerController`（基于 `QTcpSocket`）
- 结果采集：
  - `HardwareResultCollector`（信号 `dataReceived(QByteArray)`、`fileSaved(QString path)`）
  - `OfflineFileProcessor`
  - `FilePersister`（含命名规则）
- 入库：
  - `InteractionLogService`、`ResultPersistenceService`
  - 数据库实体 `InteractionLog`、`HardwareResult`（Qt SQL 模型）
- 必选 1 个核心算法（≤30 行 C++）：**硬件交互可靠性算法**（超时重试 + 心跳保活 + 失败回退）。

### (4) 用例描述（PlantUML 用例图）
- 参与者：操作员、电源/受控硬件、数据库、文件系统、上层业务模块。
- 用例：控制电源开机、接收电源状态、保存硬件返回结果为文件、离线处理文件、交互过程入库、结果入库。
- 节点 ≤12。

### (5) 界面设计（HTML，严格套用附录 B 模板）
- 三个子模块的界面可在同一主窗口下用 `.qt-tabs` 切换，也可分别给出 3 段 HTML（统一骨架）。
- 工具栏建议按钮：【连接】【开机】【开始采集】【停止】【加载离线文件】【批量处理】【刷新日志】。
- 中央工作区：
  - 电源控制：左侧 `.qt-group "通信配置"`，中部 `.qt-group "当前状态"`（电压/电流/温度 + `.qt-led`），右侧 `.qt-group "操作"` 按钮组。
  - 结果采集：顶部 Tab（在线 / 离线），下方表格 + 进度条 + 日志区。
  - 入库：顶部筛选条 `.qt-row`（时间范围、关键字、级别下拉、【检索】【清除】【重置筛选】按钮），中部 Tab（交互日志 / 结果记录）+ `.qt-table`。
- 状态栏：【硬件连接】`.qt-led` + 【实时丢包率】+ 【数据库连接】`.qt-led`。
