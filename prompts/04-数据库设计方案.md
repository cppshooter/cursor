## 二、章节定位与篇幅

**目标页数 5~7 页**。本章给出库选型、表结构、ER 关系、运维约束，不写 C++ 代码、不写界面。

## 三、写作铁律

1. 仅围绕五大功能模块产生的数据建表，**不引入用户表、权限表、角色表、菜单表等需求外实体**。
2. 数据库兼容银河麒麟 V10。推荐 **SQLite** 作为主选（嵌入式、零部署），明列备选 **达梦 DM8 / 人大金仓 KingbaseES**。
3. 数据库访问层统一表述为 "**Qt SQL 模块**（`QSqlDatabase`/`QSqlQuery`/`QAbstractTableModel`）"，不再提及 JDBC/HikariCP/MyBatis 等 Java 栈词汇。
4. ER 图与组件图使用 PlantUML，节点 ≤20。
5. 遵守附录 A 的禁用词与术语统一。

### 4.3 数据安全与运维设计
- 备份策略：定时全量 + 增量（一段话，不展开为子系统）。
- 完整性：主外键、唯一、非空约束。
- 校验：呼应安全性需求"输入信息合法性检查"。
- 审计：通过 `t_sys_log` 满足日志 ≥3 种操作。
- **容量与性能**：对日志/交互/结果类高增长表给出容量估算（行数/磁盘量级）与控制手段（复合索引、分页、按时间归档清理）；单机 SQLite 不做分库分表，超量时切换达梦/金仓的策略一句话说明（对齐 bid-type 第 4 项"容量/性能"）。
- 运维：历史日志按时间归档/清除；慢查询监控（简述）。
