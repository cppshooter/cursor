# 第4章「数据库设计方案」章节生成提示词

## 一、角色设定

你是一名资深数据库设计师，请基于本提示词与系统需求，生成《系统建设方案》第 4 章「数据库设计方案」的完整内容。

## 二、上下文输入

- `系统需求.md`：尤其关注五大功能模块所需入库的数据
- `大纲.md`：本章子节
- `项目管理.md`：GJB 438C-2021、麒麟 V10 部署约束

## 三、写作铁律

1. **严禁突破需求边界**：仅围绕五大功能模块产生的数据建表，不引入"用户表、权限表、角色表、菜单表"等需求未提到的实体。
2. 数据库选型必须兼容银河麒麟 V10（推荐：达梦 DM8 / 人大金仓 KingbaseES / PostgreSQL；明确给出选型理由与备选方案）。
3. ER 图、关系图统一用 PlantUML 表达，简单明了。
4. 全文简体中文。

## 四、本章节须覆盖的小节

### 4.1 数据库总体架构设计
- 总体架构：单库多 Schema 或单库单 Schema（推荐单库单 Schema，简单可维护）。
- 选型说明：数据库产品（达梦 / 金仓 / PostgreSQL 备选），驱动、连接池（HikariCP）。
- 部署形态：本地化部署在麒麟 V10 服务器，与应用同机或邻机部署。
- 字符集：UTF-8；时区：本地时区。
- 用 PlantUML 组件图描述 应用 ↔ ORM ↔ JDBC ↔ 数据库 的层次。

### 4.2 核心业务数据表设计

按功能模块组织（严格 5 类，不增不减）：

#### 4.2.1 任务管理相关表
- `t_task`（任务表：任务 ID、来源软件、内容、状态、接收时间、完成时间）
- `t_task_instruction`（任务分解后的指令表：任务 ID、指令序号、指令内容、状态）
- `t_task_report`（结果上报记录：任务 ID、上报内容、上报时间、目标软件、状态）
- `t_tech_state_report`（技术状态上报记录）

#### 4.2.2 数据处理相关表
- `t_auto_process_log`（自动处理过程日志）
- `t_db_table_meta`（自动生成/读取的业务表元数据登记）
- `t_table_compare_result`（表比对结果）
- `t_anomaly_alarm`（异常数据告警记录）
- `t_manual_process_log`（手动处理记录）
- 文件名解析结果、Excel 导出记录可作为字段或附属表（不新增需求外能力）

#### 4.2.3 硬件交互相关表
- `t_hw_interaction_log`（硬件交互过程入库）
- `t_hw_result`（硬件返回结果入库）
- `t_hw_offline_file`（离线处理过的文件清单）

#### 4.2.4 结果评估相关表
- `t_eval_reception`（接收效果评判记录）
- `t_timing_list`（时序列表主表）
- `t_timing_list_item`（时序项明细，含手动调整字段：modified_by、modified_at）

#### 4.2.5 系统管理相关表
- `t_hw_status_log`（硬件工作状态日志）
- `t_hw_self_test`（自检结果）
- `t_sys_metric_log`（磁盘 / CPU 等系统状态采样，对应"≥2 种"性能要求）
- `t_strategy`（策略库主表）
- `t_strategy_param`（策略生成的控制参数）
- `t_hw_config`（硬件配置信息）
- `t_sys_log`（系统日志，支持检索/清除/重置筛选 ≥3 种操作）

每张表给出：表名、用途、关键字段（字段名 / 类型 / 是否必填 / 说明）、主键、关键索引。可用 Markdown 表格呈现。

提供一张 **PlantUML ER 图** 串联五类表的主要关系（节点 ≤20，简洁）。

```plantuml
@startuml
entity t_task { ... }
entity t_task_instruction { ... }
t_task ||--o{ t_task_instruction
@enduml
```

### 4.3 数据安全与运维设计
- **备份策略**：定时全量 + 增量备份（仅描述方法，不展开为独立子系统）。
- **数据完整性**：主外键约束、唯一约束、非空约束。
- **数据校验**：呼应安全性需求「输入信息的合法性检查」，在数据入库前进行字段校验。
- **审计**：通过 `t_sys_log` 记录关键操作（与"日志操作 ≥3 种"对齐）。
- **运维**：表清理策略（历史日志归档 / 清除）、连接池配置、慢 SQL 监控（简述）。
