# 第2章「总体设计方案」章节生成提示词

## 一、上下文输入（必须随本提示词一起提供）

- `系统需求.md`、`项目管理.md`、`大纲.md`
- `_共享_写作规范.md`（附录 A）

## 二、章节定位与篇幅

本章给出本系统的架构主轴。**目标页数 6~8 页**，含 1~2 张 PlantUML 图、若干表格。不写业务代码，不写界面。

## 三、本章节小节与写作要点

### 2.1 系统设计原则
- 用表格罗列 5 条：模块化与"高内聚低耦合"、可裁剪、健壮性（输入校验 + 异常处理）、安全性（合法性校验 + 误操作二次确认）、标准化（GJB 438C-2021）。
- 每条 ≤30 字，不展开论证。

### 2.2 系统总体技术架构
- 采用 **C++17 + Qt 5.15 LTS（Widgets，Fusion 风格）** 的单机桌面架构。
- 逻辑分层：
  1. 界面层（Qt Widgets，`QMainWindow` + 多 `QDockWidget`）
  2. 业务层（五大模块的 Manager/Service 类）
  3. 数据访问层（Qt SQL，`QSqlDatabase`/`QSqlQuery`）
  4. 硬件交互层（QSerialPort / QTcpSocket）
  5. 存储层（数据库 + 文件系统）
- 用一张 **PlantUML 组件图**（≤12 节点）展示：界面层 → 业务层（5 模块） → 数据访问 / 硬件交互 → 数据库 / 文件 / 硬件。
- 用一张 **PlantUML 部署图**（≤8 节点）：上位机（银河麒麟 V10）+ 电源/受控硬件 + 上层任务软件。
- 数据主流向写成一行：任务接收 → 指令分解 → 硬件控制 → 结果采集 → 数据处理 → 结果评估 → 结果上报。

### 2.3 设计约束符合性设计

#### 2.3.1 架构专项设计
- 模块化：5 个一级业务模块独立 Qt 库或源代码目录，通过接口（抽象基类 + 信号槽）通信。
- 可裁剪：按需求拆分子模块，便于按工作日里程碑分批交付。

#### 2.3.2 开发语言选型适配
- **C++17**（编译器 GCC/Clang，由银河麒麟 V10 自带或源安装）。
- **Qt 5.15 LTS**：开源 LGPL，跨平台、Linux 支持稳定，麒麟 V10 自带或可通过软件源安装。
- 配套：构建工具 qmake 或 CMake；单元测试 QtTest；静态检查 cppcheck/clang-tidy；编码规范 Google C++ Style + 内部约束。

#### 2.3.3 部署环境适配设计
- 操作系统：银河麒麟 V10（强约束）。
- 数据库选型：推荐 **SQLite**（嵌入式，零部署，适合单机），备选 **达梦 DM8** 或 **人大金仓 KingbaseES**（信创要求场景）。
- 中间件：本地化，无外部依赖。
- 字符集：UTF-8；时区：本地时区。
- 用表格列出各运行依赖（Qt 模块、数据库、串口库）及版本范围。

### 2.4 核心技术选型与应用
用一张表，不写排比段。表格列：技术项 / 选型 / 版本 / 用途。

| 技术项 | 选型 | 版本 | 用途 |
|---|---|---|---|
| 语言 | C++ | C++17 | 全栈实现 |
| GUI 框架 | Qt Widgets（Fusion 风格） | 5.15 LTS | 桌面 UI |
| 构建 | qmake / CMake | - | 工程构建 |
| 数据库 | SQLite / DM8 / KingbaseES | 3.x / DM8 / V8R6 | 业务数据持久化 |
| ORM/SQL | Qt SQL 模块 | 随 Qt | 数据库访问 |
| 串口 | QSerialPort | 随 Qt | 硬件串口通信 |
| 网络 | QTcpSocket / QTcpServer | 随 Qt | 与上层/硬件 TCP 通信 |
| Excel | QXlsx | 1.4+ | 自动生成 Excel |
| 日志 | spdlog 或 QLoggingCategory | 1.x | 日志检索/清除/重置筛选 |
| 单元测试 | QtTest | 随 Qt | 单元测试 |

### 2.5 执行标准与规范体系
- 用表格列出 4 类标准：设计说明（GJB 438C-2021）、编码规范（含注释率 ≥30%）、文档规范（甲方模板）、配置管理规范。
- 每行 ≤40 字。
