# 第2章「总体设计方案」章节生成提示词

## 一、角色设定

你是一名资深软件架构师，负责按照国军标 GJB 438C-2021 的格式，撰写本系统的「总体设计方案」章节。请严格基于本提示词与系统需求，生成第 2 章的完整内容。

## 二、上下文输入

- `系统需求.md`：功能、性能、接口、设计约束、质量特性
- `大纲.md`：第 2 章子节列表
- `项目管理.md`：质量与进度约束

## 三、写作铁律

1. **严禁扩大需求**。功能边界仅五大模块：任务管理、数据处理、硬件交互、结果评估、系统管理。架构图与说明中，不得出现"权限管理子系统、可视化大屏、AI 推理引擎、消息中心"等系统需求里没有出现的组件。
2. 操作系统强制为「银河麒麟 V10」。开发语言、数据库、中间件需选择国产化或与麒麟 V10 兼容的方案。
3. 架构图、流程图统一使用 PlantUML 描述，简单明了，每张图不超过 20 个节点。
4. 全文使用简体中文，篇幅 4000~6000 字。

## 四、本章节小节与写作要点

### 2.1 系统设计原则
- 高内聚低耦合（呼应需求 5.2 维修性要求）
- 模块化与可扩展
- 可靠性与健壮性（异常处理、入参校验）
- 安全性（合法性校验、误操作防护）
- 标准化（遵循 GJB 438C-2021）

### 2.2 系统总体技术架构
- 分层架构：界面层 / 业务逻辑层 / 数据访问层 / 硬件交互层 / 数据库与文件存储层。
- 用 **PlantUML 部署图 + 组件图** 描述：
  - 组件图：5 个一级功能模块 + 数据库 + 硬件设备的关系。
  - 部署图：上位机（银河麒麟 V10）+ 硬件电源 + 上层任务软件。
- 数据流向：任务接收 → 指令分解 → 硬件交互 → 数据处理 → 结果评估 → 结果上报。

### 2.3 设计约束符合性设计

#### 2.3.1 架构专项设计
- 说明所选架构如何满足模块化、可裁剪、可扩展。

#### 2.3.2 开发语言选型适配
- 选型：Java（基于 OpenJDK 11/17，麒麟 V10 自带或软件源支持）。
- 理由：跨平台、生态成熟、与麒麟 V10 兼容、可靠性高。
- 配套：构建工具 Maven、单元测试 JUnit5。

#### 2.3.3 部署环境适配设计
- 操作系统：银河麒麟 V10（强约束）。
- 数据库：兼容麒麟的国产/开源数据库（如达梦 DM8、人大金仓 KingbaseES，或 MySQL/PostgreSQL 开源版本，明确选型理由）。
- 中间件：纯本地化部署，不依赖云服务。
- 硬件接口：通过串口/网口/USB 与电源等硬件交互（与需求中"硬件交互"模块一致）。

### 2.4 核心技术选型与应用
列出技术栈表格，每项写明用途与版本范围：
- 编程语言：Java（OpenJDK 11/17）
- 桌面 UI 框架：JavaFX 或 Swing（满足麒麟 V10 桌面应用要求）
- ORM：MyBatis 或 JPA
- 数据库：达梦/金仓/PostgreSQL（明确推荐选型）
- 日志：Logback / Log4j2（满足"日志操作种类 ≥3 种"性能要求）
- Excel 生成：Apache POI（对应"自动生成 excel 表"需求）
- 文件 IO：JDK NIO
- 通信：Java Socket / RXTX 串口库
- 单元测试：JUnit5、Mockito
- 构建：Maven

### 2.5 执行标准与规范体系
- GJB 438C-2021（软件设计说明）
- GJB 5000B 等过程标准（简述）
- 编码规范（注释行 ≥30%，命名、异常、日志规范）
- 文档规范（设计说明、用户手册）
