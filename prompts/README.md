# 系统建设方案 · 提示词包（Prompt Pack）

本目录是面向 LLM 的"分章节提示词集合"，用于驱动模型按《大纲.md》依次生成《系统建设方案》全文。

## 一、设计原则

1. **严格不扩大需求**：所有提示词均以 `系统需求.md` / `项目管理.md` 原文为唯一需求来源，禁止引入需求中未出现的子系统、功能或质量项。
2. **逐章生成**：每个章节一个独立的提示词文件，便于分次调用模型，逐章审核与回滚。
3. **功能模块"一模块一文件"**：第 3 章「功能模块详细设计」按系统需求中的 5 个一级功能模块拆分为 5 个独立提示词文件。
4. **子模块统一五小节**：功能模块下的每个子模块固定输出五小节——功能模块描述、操作步骤（含 PlantUML 时序图）、类/算法设计（Java）、用例描述（PlantUML 用例图）、界面设计（HTML）。

## 二、文件结构

```
prompts/
├── README.md                                  # 本文件
├── 01-引言.md                                  # 对应大纲第 1 章
├── 02-总体设计方案.md                          # 对应大纲第 2 章
├── 03-功能模块详细设计/                        # 对应大纲第 3 章（拆为 5 个模块）
│   ├── 3.1-任务管理模块.md
│   ├── 3.2-数据处理模块.md
│   ├── 3.3-硬件交互模块.md
│   ├── 3.4-结果评估模块.md
│   └── 3.5-系统管理模块.md
├── 04-数据库设计方案.md                        # 对应大纲第 4 章
├── 05-性能与接口设计方案.md                    # 对应大纲第 5 章
├── 06-通用质量特性设计方案.md                  # 对应大纲第 6 章
├── 07-质量控制方案.md                          # 对应大纲第 7 章
├── 08-项目实施与进度计划.md                    # 对应大纲第 8 章
└── 09-售后服务与保障方案.md                    # 对应大纲第 9 章
```

## 三、功能模块拆解（来自《系统需求.md》原文，严禁扩大）

| 一级模块 | 子模块拆分 | 依据条目 |
|---|---|---|
| 3.1 任务管理 | 3.1.1 任务接收 / 3.1.2 结果报送 | 任务管理 (1)(2) |
| 3.2 数据处理 | 3.2.1 自动处理 / 3.2.2 手动处理 | 数据处理 (1)(2) |
| 3.3 硬件交互 | 3.3.1 电源控制 / 3.3.2 硬件结果采集与保存 / 3.3.3 交互过程与结果入库 | 硬件交互 6 项原子能力 |
| 3.4 结果评估 | 3.4.1 硬件结果接收与评判 / 3.4.2 时序列表管理 / 3.4.3 算法调用与结果展示 | 结果评估 5 项原子能力 |
| 3.5 系统管理 | 3.5.1 状态监控（含日志） / 3.5.2 策略管理 / 3.5.3 信息管理 | 系统管理 (1)(2)(3) + 性能要求 |

## 四、子模块输出规范（约定，所有第 3 章提示词共用）

| 小节 | 要求 |
|---|---|
| (1) 功能模块描述 | 紧扣需求原文条目，列出输入/输出/依赖 |
| (2) 操作步骤 | 编号步骤 ≤10 条，含 **PlantUML 时序图**（参与者 ≤5，简单明了） |
| (3) 类 / 算法设计 | **Java 代码**仅描述架构（类/方法签名）或核心算法（≤30 行） |
| (4) 用例描述 | **PlantUML 用例图**，节点 ≤12，简单明了 |
| (5) 界面设计 | **HTML 片段**（内联 CSS），结构清晰即可，无需可运行 |

## 五、使用方式

1. 顺序读取 `01` ~ `09` 提示词文件，将其内容连同 `系统需求.md`、`大纲.md`、`项目管理.md` 一并提供给 LLM。
2. 每次只生成一个章节，便于审核。
3. 第 3 章的 5 个模块按顺序逐个生成，每次都强调"严禁扩大需求"。
4. 全部章节生成完成后，按大纲顺序拼接，形成《系统建设方案》全文初稿。

## 六、统一约束（适用于所有章节）

- 操作系统：银河麒麟 V10（强约束）。
- 标准：GJB 438C-2021。
- 维护期：5 年。
- 验收截止：2026-08-31。
- 性能要求：日志操作 ≥3 种、系统状态信息显示 ≥2 种。
- 接口范围：仅控制指令、状态信息、指令执行结果三类。
- 注释率：源代码注释行 ≥ 总代码行 30%。
- 文档语言：简体中文。
