# 系统建设方案 · 提示词包（Prompt Pack）

本目录是面向 LLM 的"分章节提示词集合"，用于驱动模型按《大纲.md》依次生成《系统建设方案》全文。

## 一、设计原则

1. **严格不扩大需求**：所有提示词以 `系统需求.md` / `项目管理.md` 原文为唯一需求来源，不引入需求中未出现的子系统、功能或质量项。
2. **章节完整性以 bid-type 为准**：本方案属软件开发类，各章节结构与强制完整性须满足 `knowledge/bid-type/software.md`（软件类技术方案撰写要点），尤其：系统总体技术架构四项必含（逻辑分层图 / 数据主流向图 / 组件图 / 部署图），功能模块逐模块五要素齐全，数据库设计 + 接口设计齐备。
3. **逐章生成**：每章一个独立提示词文件，便于分次审核与回滚。
4. **功能模块"一模块一文件"**：第 3 章按系统需求中的 5 个一级功能模块拆分为 5 个独立提示词文件。
5. **子模块统一五小节**：功能模块描述 / 操作步骤（PlantUML 时序图）/ 类与算法设计（C++/Qt）/ 用例描述（PlantUML 用例图）/ 界面设计（HTML，统一 Qt 样式），对应 `knowledge/bid-type/software.md` 功能模块五要素。
6. **统一风格、统一样式、统一术语**：通过两个共享附录强制约束（见下文）。

## 三、全局约束（每次调用 LLM 必须随提示词一起提供）

每次生成任意章节时，**至少提供以下文件作为上下文**：

1. `系统需求.md`（需求来源）
2. `knowledge/bid-type/software.md`（软件类章节强制要求，判定本章是否"缺项"的依据）
3. `_共享_写作规范.md`（附录 A）
4. `_共享_界面样式规范.md`（附录 B，仅第 3 章的子模块界面设计需要）
5. 待生成章节对应的提示词文件（如 `02-总体设计方案.md`）

> 生成后必须对照 `knowledge/bid-type/software.md` 的"强制完整性"与"常见失分点"自检；如本章对应的强制项（如总体架构四图、功能模块五要素）缺失，须补齐后再交付。

## 四、统一技术栈与约束

| 约束 | 取值 |
|---|---|
| 操作系统 | 银河麒麟 V10 |
| 开发语言 / GUI 框架 | C++17 + Qt 5.15 LTS（Widgets，Fusion 风格） |
| 构建工具 | qmake 或 CMake |
| 数据库 | SQLite（嵌入式）/ 达梦 DM8 / 人大金仓 KingbaseES（任选一项推荐） |
| 数据库访问 | Qt SQL（QtSql 模块，QSqlDatabase / QSqlQuery） |
| 串口/网络 | QSerialPort / QTcpSocket / QTcpServer |
| 日志 | spdlog 或 Qt 日志框架 |
| Excel 导出 | QXlsx（开源 Qt 库） |
| 单元测试 | Qt Test (QtTest) |
| 标准 | GJB 438C-2021 |
| 维护期 | 5 年 |
| 验收截止 | 2026-08-31 |
| 性能指标 | 日志操作 ≥3 种、系统状态信息显示 ≥2 种 |
| 接口范围 | 控制指令、状态信息、指令执行结果 三类 |
| 注释率 | 源代码注释行 ≥ 总代码行 30% |
| **正文总篇幅** | **≤80 页（A4，小四，1.5 倍行距估算）** |
| 文档语言 | 简体中文 |
| 行文风格 | 工程化、克制、不用 AI 味油腻词（详见附录 A） |

## 五、功能模块拆解（来自《系统需求.md》原文，严禁扩大）

| 一级模块 | 子模块 | 依据 |
|---|---|---|
| 3.1 任务管理 | 任务接收 / 结果报送 | 任务管理 (1)(2) |
| 3.2 数据处理 | 自动处理 / 手动处理 | 数据处理 (1)(2) |
| 3.3 硬件交互 | 电源控制 / 硬件结果采集与保存 / 交互过程与结果入库 | 硬件交互 6 项原子能力 |
| 3.4 结果评估 | 接收与评判 / 时序列表管理 / 算法调用与结果展示 | 结果评估 5 项原子能力 |
| 3.5 系统管理 | 状态监控（含日志） / 策略管理 / 信息管理 | 系统管理 (1)(2)(3) + 性能要求 |

## 六、子模块输出规范（第 3 章 5 个模块共用）

| 小节 | 要求 |
|---|---|
| (1) 功能模块描述 | 紧扣需求原文条目，列出输入/输出/依赖。不写空泛"概述" |
| (2) 操作步骤 | **以软件用户操作手册风格撰写**：明确顶部菜单（如"文件→新建任务"）、工具栏按钮、表单字段、表格列、对话框、快捷键。编号步骤 ≤10 条，紧跟一张 PlantUML 时序图（参与者 ≤5）|
| (3) 类与算法设计 | **C++17 + Qt 代码**（头文件签名 / 关键方法 / 信号槽 / 30 行内核心算法），不写完整业务实现 |
| (4) 用例描述 | PlantUML 用例图，节点 ≤12，简单明了 |
| (5) 界面设计 | **必须复用附录 B 的 HTML 模板骨架与 `.qt-*` 类名**，体现 Qt Fusion 风格 |
